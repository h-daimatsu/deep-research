# 共通骨格

レポート構造を動的に設計する場合でも、以下の要素は必ず含める。これらは全テーマに共通する骨格であり、削除・省略できない。

## 必須要素と配置順序

```
**作成日**: YYYY-MM-DD JST    ← レポート冒頭（タイトル直下）に固定
## エグゼクティブサマリー          ← 作成日の直後に固定
  （本体セクション群）             ← 動的に設計する部分
## 情報の矛盾・不確実な点          ← 本体の後に固定
## 追加調査が必要な領域            ← 上記の後に固定
## 結論                           ← 末尾近くに固定
## 出典                      ← 最末尾に固定
```

動的に設計するのは「本体セクション群」の部分のみ。共通骨格の要素の順序は変更できない。

## 各要素の仕様

### 作成日

レポートのタイトル直下、エグゼクティブサマリーの前に `**作成日**: YYYY-MM-DD JST` の形式で記載する。日付は調査開始時のローカル日付（JST）を使用する。

## 各要素の仕様

### エグゼクティブサマリー

レポートの冒頭に置く。以下を簡潔に提示する:

- 中心主題（レポート全体を貫く問い）
- 最も重要な発見（2〜3点）
- 結論の要旨（1〜2文）

読者がエグゼクティブサマリーだけで全体像を把握できることを目指す。ただし詳細は本体セクションに委ね、ここでは要約に徹する。

### 情報の矛盾・不確実な点

調査中に発見した以下を記録する:

- ソース間で矛盾する情報（どのソースが何を主張しているかを明記）
- 確認できなかった事項
- 信頼性が低いが無視できない情報
- データの欠損や古さ
- **矛盾の有無に関係なく低確信な柱の主張**: 結論を支える主張のうち、(a) 単一源泉（源泉を畳むと独立源泉が1つ＝偽のコンセンサスを含む）、(b) 事実抽出系（正確な財務数値・議決権比率・契約文言等）でありながら一次未確認、のいずれかに該当するものは、ソース間で矛盾していなくても必ずここに列挙する。**ただし、集合知・コンセンサス系（アナリストコンセンサス、市場規模推計等）と解釈・市場観系の主張は、複数の独立ソースが一致していれば「二次情報のみ」であっても低確信扱いにしない**（`workflow.md` の「主張のタイプ別の確からしさ戦略」を参照）。これらをここに列挙するのは独立源泉が実質1つの場合に限る

このセクションは、レポートの信頼性を読者が判断するための材料を提供する。「矛盾がなかった」場合でも、不確実な点や情報の限界を記載する。確信度の明示には `citation-format.md` の確信度タグを用いる。

### 追加調査が必要な領域

本調査の範囲では十分にカバーできなかった論点を列挙する:

- 時間的制約で深掘りできなかったテーマ
- 一次情報にアクセスできなかった領域
- 今後の動向次第で結論が変わりうる論点

各項目に「なぜ追加調査が必要か」の理由を付す。

### 結論

`conclusion-guide.md` の5要素を必ず含める:

1. **主題への回答**: 中心主題に対する最終的な答え
2. **主要な発見の再提示**: 最も重要な2〜3点の短い再確認
3. **含意（So what）**: 誰にとって、何を意味するか
4. **推奨事項・注視ポイント**: 読者が取るべきアクション、または注視すべき先行指標
5. **展望**: 短期〜中期で最も蓋然性の高いシナリオ

段落形式の散文で統合的に記述する。箇条書きで5要素を並べない。判断を完全に読者へ委ねる結び方は避ける。

### 出典

`citation-format.md` の脚注定義形式で、本文中の `[^N]` に対応する定義を列挙する。

```markdown
## 出典

[^1]: [タイトル](URL) — 発行元 (日付)

[^2]: [タイトル](URL) — 発行元 (日付)
...
```
